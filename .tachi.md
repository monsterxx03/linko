# Linko - Tachi Context

Linko is a transparent MITM proxy for LLM traffic analysis on macOS. It intercepts HTTPS traffic using macOS firewall rules (pfctl) and provides real-time visualization of LLM API conversations.

## Build Commands

```bash
# Install dependencies
make deps          # Go dependencies
make ui-deps       # UI dependencies (uses bun)

# Build
make build         # Build Go binary to bin/linko
make ui-build      # Build React UI to pkg/ui/dist
make all           # Build UI + Go binary

# Development
make ui-dev        # Start Vite dev server (http://localhost:5173)

# Code quality
make fmt           # Format code (goimports)
make lint          # Lint with golangci-lint

# Testing
make test          # Run all tests
make test-coverage # Run tests with coverage

# Single package test
go test -v ./pkg/mitm/llm/... -run TestAnthropic
go test -v ./pkg/dns/... -run TestCache
```

### Key Design Patterns

**MITM Traffic Flow:**
1. `pkg/proxy/transparent_darwin.go` sets up pf rules to redirect port 443 → local proxy
2. `pkg/proxy/mitm_handler.go` handles TLS handshake with generated certs
3. `pkg/mitm/http_processor.go` parses HTTP request/response
4. `pkg/mitm/llm_inspector.go` detects LLM API calls and routes to appropriate parser
5. Events published to `EventBus`, consumed by admin SSE endpoints and TUI

**DNS Splitting:**
- `pkg/dns/splitter.go` uses `pkg/ipdb/china_ip.go` (cidranger) to determine if IP is domestic
- Domestic queries → AliDNS/114DNS; Foreign queries → Google/Cloudflare via TCP

**Event System:**
- `pkg/mitm/event_bus.go` provides pub/sub for real-time traffic events
- Admin server exposes SSE endpoints (`/api/mitm/traffic/sse`, `/api/llm/conversation/sse`)
- TUI (`cmd/linko/traffic/`) connects via SSE for live monitoring

## Development Notes

**Running MITM mode:**
```bash
# Generate CA cert first
linko gen-ca
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ~/.config/linko/certs/ca.crt

# Run MITM proxy (requires sudo for firewall rules)
sudo go run ./cmd/linko mitm --whitelist "api.anthropic.com"

# Run TUI (connects to admin server)
go run ./cmd/linko tui
```

**Testing MITM with curl:**
```bash
curl -v https://api.anthropic.com  # Should show cert issued by "Linko MITM CA"
```

**Using with Claude Code:**
```bash
NODE_TLS_REJECT_UNAUTHORIZED=0 claude
```

## LLM Parser Implementation

Located in `pkg/mitm/llm/`. Each provider implements:
- Request parsing (model, messages, tools)
- Response parsing (content, streaming deltas, thinking)
- Provider detection via URL patterns

**Supported APIs:**
- OpenAI Chat Completions (`/v1/chat/completions`)
- Anthropic Messages (`/v1/messages`)
- Google Gemini (`/v1beta/models/...:generateContent`)

**Key Types:**
- `types.go` - Common LLM event structures
- `provider.go` - Provider detection interface
- Streaming responses use `sse_inspector.go` for delta accumulation
